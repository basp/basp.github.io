two dimensional discrete convolution

January 14, 2018

Today I’m (finally) able to introduce something concrete: 2D convolutions for .NET (i.e. linear filtering). I just wrote this quickly so it’s not that optimized and probably needs some more love but it’s usuable and it works.

I tried to dig into convolutions from a more general perspective but they are (no pun intended) quite convoluted. So we’re going to limit ourselves to a very particular case of convultions.

You probably have seen these kinds of convolutions in action because they are basically equivalent to 3D shaders. If you ever programmed or saw for instance a blur effect then you have seen a 2D discrete convolution in action.

the basics

In theory the 2D convolution is simple but hard to explain. We assume a source image (a Matrix<T> with some kind of values of type T) and a kernel (a Matrix<double> that is also called weights).

We need to loop through every pixel in the source image:

for(var row = 0; row < image.RowCount; row++)
{
    for(var col = 0; col < image.ColumnCount; col++)
    {
        ...
    }
}

Now for every pixel in the source image we place the center of our kernel at the position of row and col. Then we loop through every weight in the kernel as it is relative to our source image position and apply our accumulator function to all those values to produce a new value for the row and col position in our target image. This requires kernel.RowCount % 2 = 1 and kernel.ColumnCount % 2 = 1. In other words, a kernel where the RowCount and ColumnCount are uneven.

We also need to loop through every weight in the kernel so this means even more loops. In the end we end up with a total of four nested loops to handle every combination of source pixel and kernel value.

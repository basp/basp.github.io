random thoughts

April 5, 2018

This is just a stream of random statements. If you agree, great! If you disagree that’s cool too.

  • To do C right you will have to know assembly
  • So C is a somewhat leaky abstraction
  • C++ is just a bad superset of C
  • Some people write C++ code
  • Dynamic programming languages are super awesome
  • Until they are not
  • Static types are a nuisance
  • Until they offer you guidance
  • Live code updates are awesome (love and learn from Erlang)
  • Functional languages are cool
  • Dynamically strong typed functional languages are even cooler
  • But you’ll end up adding type specifications anyway
  • It’s reasonable to expect more static constructs as projects grow
  • That’s why things like TypeScript and gradual typing are exciting
  • Haskell is the future
  • But sadly nobody understands monads and functors and why they are great
  • And those who do seem to be unable to explain it to mere mortals
  • Expressions are to be preferred above statements almost all the time
  • Unless you absolutely need a statement
  • It’s a bit of a pain to change state in some languages for good reasons
  • An expression should not change state
  • A statement should not return a value
  • Sequences (think lazy lists or enumerables/iterables) are good
  • Think twice or even thrice about every if statement you write
  • That accounts for switch statements as well
  • Logic should ideally flow in one straight line via expressions and then via the stack back to the caller (branching statements break this logic and should be comparable to the evil of GOTO statements)
  • If try to write everything in sequence logic you won’t need much
  • The sun is bright don’t directly look into it

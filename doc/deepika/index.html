 (require deepika/db) package: deepika

The database module is responsible for persisting the state of the world. In our case, the world consists mainly of three things: objects, properties and verbs.

value

objid : objid?

An object id is just a simple wrapper around an integer. We need these to keep them apart from regular integer values.

value

$nothing : nothing?

Represents the absence of a value and can generally be used as a sensible default for database values.

procedure

(nothing? x) → boolean?

  x : any/c
Returns #t if x is $nothing.

procedure

(objid? x) → boolean?

  x : any/c
Returns #t if x is an object id and #f otherwise.

Note that the result of this function only confirms the data type. It does not confirm the existance of a valid object. Use valid? for that case instead.

Converts object id oid into an integer number.

procedure

(number->objid num) → objid?

  num : integer?
Converts number num into an object id.

procedure

(valid? x) → boolean?

  x : any/c
Returns #t if x is an object id that points to an actual object and #f otherwise.

Examples:
> (define oid (number->objid 1))
> (valid? oid)

#f

> (create-object!)

'#s(objid 1)

> (valid? (number->objid 1))

#t

procedure

(valid+? x) → boolean?

  x : any/c
Like valid? but it will also return #t if x is $nothing.

Examples:
> (valid? $nothing)

#f

> (define oid (create-object!))
> (valid? oid)

#t

> (valid? (objid 123))

#f

procedure

(create-object! pid) → objid?

  pid : valid+?
Create a new object. If pid is not specified then the object’s parent property will be set to $nothing.

procedure

(destroy-object! oid) → any/c

  oid : valid?
Destroy the object with id oid. It will no longer be accessible from the database.

procedure

procedure

(get-parent oid) → valid+?

  oid : valid?
Returns the oid of the parent of the object referenced by object oid.

procedure

(set-parent! oid new-parent) → any

  oid : valid?
  new-parent : valid+?
Sets the parent of the object referenced by object id oid to the new parent specified by the new-parent object id.

procedure

(get-children oid) → (listof valid?)

  oid : valid?
Returns the objects that have oid set as their parent object.

Examples:
> (define o (create-object!))
> (define p (create-object!))
> (get-parent o)

'#s(objid -1)

> (set-parent! o p)
> (get-parent o)

'#s(objid 4)

> (get-children (get-parent o))

'(#s(objid 3))

vectors¶

A vector is a row (or column depending on how you look at it) of numbers. If we want to write a literal vector in math it often is in written in brackets:

$\begin{bmatrix}1 & 2 & 3\end{bmatrix}$

If we wanna a particular matrix more often and don't wanna repeat the numbers each time we can give it a name. The name ${v}$ is a popular choice.

${v} = \begin{bmatrix}4 & 5 & 6\end{bmatrix}$

Sometimes we want to make it extra clear that we are dealing with a vector. There's multiple ways to do that but we'll use a notation like $\vec{v}$ for the remainder of this text.

We can also easily represent a vector in python as a list of numbers.

In [218]:
v = [1,2,3] # a vector of length 3
N = len(v)  # N = 3

In the above case we can say the length of vector ${v}$ is equal to ${N}$ which is equal to 3.

${N} = 3$

scalars¶

A vector of length one is basically a number. We can also think of it as a direction in a one-dimensional space.

This is also sometimes called a scalar which basically is a something that you can multiply with some vector to get a new vector. Note that all real numbers are valid scalars but not every scalar has to be a real number.

matrices¶

Between matrices, vectors and scalars, the matrix is actually the most general. A vector is a $1 \times {N}$ matrix and a scalar is a $1 \times 1$ matrix. They have awesome notation as well, take, for example, this matrix ${M}$:

$ M = \begin{bmatrix}1 & 2 & 3\\4 & 5 & 6\end{bmatrix}$

It might be more easy to see now how a vector is just a matrix with one dimension.

$v = \begin{bmatrix}1 & 2 & 3\end{bmatrix}$

Or as a column matrix:

$v = \begin{bmatrix}1\\2\\3\end{bmatrix}$

And a scalar can be just:

$v = \begin{bmatrix}{x}\end{bmatrix}$

Which is a 1x1 vector, a 1x1 matrix and basically equivalent to any real number ${x}$.

We can observe:

  • Every scalar is a vector (a 1x1 vector)
  • Every vector is a matrix (a 1xN matrix)
  • Not every vector is a scalar
  • Not every matrix is a vector

noise functions¶

I'm not sure if there's an official definition for a noise function but for our intends and purposes it's a function $f$ that takes a vector $\vec{x}$ of length ${N}$ and returns a floating point (decimal) number ${y}$ in the range 0 to 1.

$y = f \vec{x}$ where $0 \le {y} \le 1.0$

Now that we have defined what a noise function is we can actually define one:

$y = f x = 0.30$

There are two things to note:

  • We got rid of $\vec{x}$ and replaced it with just ${x}$ what's up with that?
  • Our noise function $f$ is defined as a constant; how noisy is it?

We're gonna replace $\vec{x}$ with ${x}$ for now because it just makes things more simple. If we can make this work for one dimension (where the input for the noise function is basically just a ${x}$ coördinate) we should be able make it work in ${N} dimensions as well.

In [201]:
def lerp(v0, v1, a):
    return (1.0 - a) * v0 + v1 * a

The lerp function is the linear interpolation function.

The scurve3 and scurve5 functions are examples of smoothstep functions.

In [205]:
def scurve3(a):
    return a * a * (3.0 - 2.0 * a)

def scurve5(a):
    a3 = a * a * a
    a4 = a3 * a
    a5 = a4 * a
    return (6.0 * a5) - (15.0 * a4) + (10.0 * a3)

x = np.linspace(0, 1.0, 100)
y1 = lerp(0.0, 1.0, x)
y2 = scurve3(x)
y3 = scurve5(x)
plt.axis([0, 1.0, 0, 1])
plt.plot(x, y1, "--", label="lerp")
plt.plot(x, y2, label="cubic")
plt.plot(x, y3, label="quadratic")
plt.legend(loc=2)

The curve function uses the $\cos$ from 0 to $\pi$ to produce a smooth curve.

In [204]:
def curve(a):
    return (1.0 - np.cos(a * np.pi)) * 0.5

Note that it produces almost exactly the same result as the scurve3 smoothstep function.
